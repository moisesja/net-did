## Review Results

- High: log entries are not chained to the previous `versionId`, so historical entries can be rewritten after later entries already exist.
- High: resolution does not bind the requested DID/SCID to the hosted log, so wrong-SCID requests can resolve to the hosted DID anyway.
- High: witness validation is non-compliant; missing or malformed witness data is accepted, and the witness file model does not match the spec.
- Medium: the API cannot produce `did-witness.json` artifacts for witnessed operations, so witnessed DIDs are not publishable end-to-end.
- High: pre-rotation can be bypassed by omitting `updateKeys` on later entries.
- Medium: `versionId` / `versionTime` resolution semantics are wrong; missing queries fall back to latest, and later corruption blocks earlier valid versions.
- Low: method-specific metadata is incomplete; `versionTime` is not exposed in `DidDocumentMetadata`.
- Low: README and the generated conformance report overstate or contradict current `did:webvh` behavior.
