### Architecture decisions
- Reused `VerificationMethodRepresentation` from `NetDid.Core.Model` instead of creating a duplicate
- Numalgo handlers are `internal` classes orchestrated by `DidPeerMethod` (single public API surface)
- Ed25519 → X25519 public key derivation uses the standard birational map `u = (1+y)/(1-y) mod p` with BigInteger arithmetic, keeping NSec as the sole crypto dependency
- Numalgo 4 uses `DidDocumentSerializer` for JSON serialization, SHA-256 multihash for the short form
