# Issue #49 — did:webvh URL mapping accepts unsafe host/path encodings

## Branch
`fix/issue-49-webvh-url-mapping-validation`

## Vulnerability summary

`DidUrlMapper.MapToLogUrl` / `MapToWitnessUrl` percent-decode the domain
chunk with `Uri.UnescapeDataString` and string-interpolate it into an
HTTPS URL with `new Uri($"https://{host}/...")`. This lets a crafted
DID smuggle `@` (userinfo, host pivot), `/` (path injection), `\`,
control chars, or path-traversal segments through resolution.

Confirmed in `src/NetDid.Method.WebVh/DidUrlMapper.cs:15-36`.

Severity: Medium — SSRF / unintended fetch target on resolvers of untrusted DIDs.

## Plan

### Code change

**`DidUrlMapper.cs`** — centralize URL construction:

1. After `ExtractDomainAndPath`, run new `ValidateDomain(string raw, string did) -> (string host, int? port)`:
   - empty → reject
   - `Uri.UnescapeDataString`
   - reject decoded chars: `@`, `/`, `\`, `?`, `#`, space, `<`, `>`, `[`, `]`, control chars
   - split on `:` once for port; reject multiple colons (no IPv6 — spec uses encoded port only)
   - `Uri.CheckHostName(host) != UriHostNameType.Unknown`
   - port (if any): integer in [1, 65535]
2. Run new `ValidatePathSegments(string? path, string did) -> List<string>`:
   - `null` → empty list
   - split on `/` (already joined from `:` parts)
   - each segment: non-empty, after `UnescapeDataString` ≠ `.` / `..`, contains no `/ \ ? #` and no control chars
3. Build final URI with `UriBuilder`:
   ```csharp
   var b = new UriBuilder("https", host) { Port = port ?? -1 };
   b.Path = segments.Count == 0
       ? $"/.well-known/{filename}"
       : "/" + string.Join("/", segments) + "/" + filename;
   return b.Uri;
   ```
4. Throw `ArgumentException` on any validation failure (existing pattern).

**`DidWebVhMethod.cs`** — call the same validators on `createOptions.Domain` and `createOptions.Path` at the top of `CreateAsync`, so malformed inputs fail at create time too (per issue acceptance criteria).

### Tests (extend `DidUrlMapperTests.cs`)

Per the issue's "Regression tests to add":

1. `%40` in domain → rejected (`ArgumentException`).
2. `%2F` in domain → rejected.
3. `%5C` in domain → rejected.
4. Empty host → rejected.
5. Invalid port (`%3A99999`, `%3Aabc`, `%3A0`) → rejected.
6. `example.com%3A8443` still maps to `https://example.com:8443/.well-known/did.jsonl`.
7. Path with `.` segment → rejected.
8. Path with `..` segment → rejected.
9. Path with `%2E%2E` → rejected.
10. Backslash injection (`%5C`) in domain → rejected.
11. Existing positive mappings still pass.

Plus 1–2 tests for `DidWebVhMethod.CreateAsync` rejecting `Domain = "evil@host"` and `Path = "../etc"`.
