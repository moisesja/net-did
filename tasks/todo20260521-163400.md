# Issue #47 — Rust FFI RustSec maintenance/unsoundness warnings

## Branch
`fix/issue-47-rust-ffi-audit`

## Pre-fix audit (zkryptium 0.2.2)

`cargo audit` in `native/zkryptium-ffi` reported 6 allowed warnings:

| Advisory | Crate | Class | Pulled in via |
|---|---|---|---|
| RUSTSEC-2021-0139 | ansi_term 0.12.1 | unmaintained | cargo-license → zkryptium |
| RUSTSEC-2024-0375 | atty 0.2.14 | unmaintained | clap → cargo-license → zkryptium |
| RUSTSEC-2021-0141 | dotenv 0.15.0 | unmaintained | zkryptium |
| RUSTSEC-2024-0370 | proc-macro-error 1.0.4 | unmaintained | clap_derive → clap → cargo-license → zkryptium |
| RUSTSEC-2021-0145 | atty 0.2.14 | unsound | (same as above) |
| RUSTSEC-2026-0097 | rand 0.8.5 | unsound | zkryptium |

## Fix

Upgrade `zkryptium` 0.2.2 → 0.6.1. The 0.6 release removes the
cargo-license / dotenv / old-clap / atty transitives. No FFI source
changes are needed — the `BBSplusPublicKey`, `BBSplusSecretKey`,
`KeyPair`, `Signature`, `PoKSignature`, and `BbsBls12381Sha256` APIs
used in `src/lib.rs` are still present in 0.6.x with the same shape.

## Post-fix audit (zkryptium 0.6.1)

5 of 6 warnings eliminated. Dependency closure shrank from 103 → 51
crates. Only RUSTSEC-2026-0097 (`rand 0.8.5`) remains.

### Reachability of RUSTSEC-2026-0097

Advisory text: "Rand is unsound with a custom logger using `rand::rng()`".

- Neither the FFI shim (`src/lib.rs`) nor `zkryptium 0.6.x` installs a
  custom logger or invokes `rand::rng()` from one.
- `rand` is reached only through standard BBS+ key generation and signing
  (`KeyPair::generate`, `Signature::sign`). Those code paths do not
  exercise the unsound condition.
- Conclusion: not reachable in NetDid's usage.

Documented in `native/zkryptium-ffi/README.md` "Security audit".

## CI

Added a `rust-audit` job to `.github/workflows/ci.yml`:

```yaml
rust-audit:
  runs-on: ubuntu-latest
  steps:
    - uses: actions/checkout@v4
    - uses: dtolnay/rust-toolchain@stable
    - run: cargo install --locked cargo-audit
    - working-directory: native/zkryptium-ffi
      run: cargo audit --ignore RUSTSEC-2026-0097
```

Only that one advisory is ignored. Any other warning fails the job.
