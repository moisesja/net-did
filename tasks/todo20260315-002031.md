## Findings

- `#27` Fixed.
  - `ExistingKey` EC public keys are now normalized to compressed SEC1 form before DID construction.
  - `did:key` and `did:peer:0` resolution now validates both key length and EC point validity.
  - Regression tests now cover uncompressed external `ExistingKey` normalization and malformed EC-point rejection.

- `#28` Fixed.
  - `did:peer:2` verification method IDs are now relative (`#key-1`, `#key-2`, ...).
  - Explicit fragment service IDs are preserved as relative IDs.
  - Auto-generated service IDs now follow `#service`, `#service-1`, ...
  - Encoder/decoder now preserves explicit service IDs in the encoded service block.
  - Regression tests now assert the relative-ID behavior instead of the old absolute-ID behavior.

- `#25` Fixed.
  - `DidDocument.Id` is no longer required for numalgo 4 input documents.
  - Serializer omits `id` when not set, allowing spec-style pre-contextualization documents.
  - Numalgo 4 tests no longer use the placeholder-ID workflow.
  - Long-form encoding remains multibase + JSON multicodec, and `alsoKnownAs` includes the short form.
  - The earlier concern about post-hoc rewriting no longer blocks spec compliance because the input model now cleanly represents the pre-contextualization document.
